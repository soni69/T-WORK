# Telegram-бот отчётности по продажам

Бот для ежедневной отчётности по продажам с несколькими направлениями и фокусными отчётами несколько раз в день.

## Возможности

- **Несколько направлений**: по блокам (Sim, Страховки, Wink и др. — настраиваемо)
- **Дневной отчёт**: в 12:00, 16:00 и 19:00 (МСК) в чат приходит *напоминание*; сотрудник заполняет показатели в боте (ЛС) → можно добавить комментарий (или /skip) → бот спрашивает «Отправить сводку в чат?» (Да/Нет); после подтверждения сводка уходит в чат.
- **Вечерний финальный отчёт**: то же — заполнение → комментарий (по желанию) → подтверждение отправки в чат.
- **Сводка**: `/summary` или `/summary DD.MM` — итоги за день (за другую дату — по формату DD.MM; корректировка только у РТТ)
- **Планы (РТТ)**: по кнопке «Планы» — просмотр, «Корректировать направление» (одно значение на текущий месяц), «Внести планы на месяц» — пошаговый ввод по всем направлениям (YYYY-MM → значение по каждому, Skip = 0). Отмена — кнопка или /cancel
- **Роли**: РТТ (руководитель торговой точки) — корректировка планов и сводки, подтверждение доступа ПК; ПК (продавец-консультант) — только отчёты и просмотр
- **Подтверждение доступа**: новый пользователь нажимает /start → заявка попадает в список; РТТ заходит в бота, кнопка «Заявки на доступ» или /pending — подтверждает или отклоняет. После подтверждения ПК получает уведомление и может пользоваться ботом
- **Добавление отчётов**: `/report` — разовое внесение по направлению
- **Как дела?**: кнопка в меню — краткая строка: % плана по сумме и сколько направлений в норме по плану/день
- **Экспорт (РТТ)**: `/export plans [YYYY-MM]` или `/export reports [YYYY-MM]` — выгрузка в CSV-файл

## Установка

1. Python 3.10+

2. Установка зависимостей:
```bash
pip install -r requirements.txt
```

3. Создайте бота в [@BotFather](https://t.me/BotFather) и получите токен.

4. Скопируйте `.env.example` в `.env` и заполните:
```bash
copy .env.example .env
```

5. В `.env` укажите:
   - `BOT_TOKEN` — токен от BotFather
   - `REPORT_CHAT_IDS` — ID чатов/групп для отчётов (через запятую)
   - `RTT_USER_IDS` — ID пользователей с ролью РТТ (через запятую); остальные — ПК

   Файл `.env` не коммитьте в репозиторий (в проекте есть `.gitignore`).

## Запуск

```bash
python bot.py
```

## Запуск на сервере

1. Скопируйте проект на сервер, создайте виртуальное окружение и установите зависимости:
   ```bash
   python -m venv venv
   source venv/bin/activate   # Linux/macOS
   # или: venv\Scripts\activate   # Windows
   pip install -r requirements.txt
   ```

2. Создайте `.env` из примера и заполните переменные:
   ```bash
   cp .env.example .env
   # Отредактируйте .env: BOT_TOKEN, REPORT_CHAT_IDS, RTT_USER_IDS
   ```

3. Запуск в фоне (Linux, через nohup):
   ```bash
   nohup python bot.py >> bot.log 2>&1 &
   ```
   Или через systemd: создайте юнит с `ExecStart=/path/to/venv/bin/python /path/to/bot.py`, `WorkingDirectory=/path/to/project`.

4. Логи пишутся в stderr (уровень INFO). Для отладки можно временно выставить `LOG_LEVEL=DEBUG` и обрабатывать его в `config.py`/`bot.py` при необходимости.

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и список команд |
| `/report` | Добавить отчёт о продажах |
| `/summary [DD.MM]` | Сводка за день (без аргумента — сегодня; DD.MM — за другую дату) |
| `/daily` | Дневной отчёт (пошагово; в конце — комментарий и подтверждение отправки в чат) |
| `/evening` | Вечерний отчёт (пошагово; комментарий и подтверждение отправки) |
| `/plans` | Показать планы на месяц (у РТТ — с кнопками «Корректировать» и «Внести планы на месяц») |
| `/setplan` | Задать/изменить план вручную: `/setplan YYYY-MM направление сумма` (только РТТ) |
| `/export` | Экспорт в CSV: `/export plans [YYYY-MM]` или `/export reports [YYYY-MM]` (только РТТ) |
| `/pending` | Заявки на доступ — подтвердить/отклонить (только РТТ) |
| `/help` | Справка |

## Настройка

В `config.py` можно изменить:

- **SALES_DIRECTIONS** — список направлений продаж
- **REPORT_SCHEDULE** — часы фокусных отчётов (по умолчанию 12, 16, 19)
- **DATABASE_PATH** — путь к файлу SQLite

## Как узнать ID чата

1. Добавьте бота [@userinfobot](https://t.me/userinfobot) в группу — он покажет ID чата
2. Или перешлите сообщение боту [@getidsbot](https://t.me/getidsbot)

Для личного чата — ID пользователя (число).
